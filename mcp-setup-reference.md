> Быстрая справочная карта для подключения MCP-серверов к Claude Code.
> Формат: что делает, команда подключения, проверка, устранение ошибок.

## Философия этого документа

Этот файл — **quick-reference для подбора нужной команды/паттерна**. Вы не настраиваете MCP вручную по памяти — вы используете его как:

1. **Шпаргалка** — быстро найти команду подключения нужного MCP-сервера в нужный момент
2. **Reference для Claude** — загрузить в проект, чтобы Claude выбирал правильный MCP под задачу
3. **Диагностика** — когда MCP падает, здесь собраны типичные ошибки и их решения

**Принцип работы:**
- Вы — архитектор: знаете, ЧТО нужно подключить (Supabase для БД, Context7 для доков)
- Claude Code — исполнитель: знает, КАК это подключить и проверить
- Этот документ — общий язык между вами

**Как применять:**
```
Сценарий 1: Claude настраивает MCP
  Вы → "подключи Supabase MCP к этому проекту" → Claude следует правилам из этого файла

Сценарий 2: Вы настраиваете сами
  Вы → открываете этот файл → копируете нужную команду → вставляете в терминал

Сценарий 3: MCP сломался
  Claude/Вы → проверяете раздел troubleshooting → следуете золотому правилу (перезапуск сессии)
```

---

## Золотое правило

> **Если MCP-авторизация падает после 2 попыток редактирования конфига — ПЕРЕЗАПУСТИ СЕССИЮ.**
> Claude, правящий конфиг-файлы по кругу, почти никогда не чинит проблему.
> Чистая сессия с правильным конфигом работает в 90% случаев.

## Общая архитектура MCP

```
Claude Code (VS Code)
    │
    ├── MCP: Supabase ──► БД проекта (таблицы, данные, миграции)
    ├── MCP: GitHub ──► Репозитории (коммиты, PR, issues)
    ├── MCP: Context7 ──► Актуальная документация библиотек
    ├── MCP: ChatPlace ──► Telegram-боты и автоматизации
    └── MCP: NotebookLM ──► Аудио-подкасты из документации
```

MCP-серверы — это мост между Claude Code и внешними сервисами. Без них Claude работает только с локальными файлами.

## 1. Supabase MCP

### Что делает
Подключает Claude Code напрямую к PostgreSQL-базе данных проекта. Позволяет выполнять SQL-запросы, просматривать таблицы, применять миграции — всё из Claude Code.

### Подготовка
1. Откройте Supabase Dashboard: `https://supabase.com/dashboard/project/ВАШ_REF`
2. Скопируйте **Project Ref** из URL (например: `abc123example`)
3. Создайте **Access Token**: Dashboard → Account → Access Tokens

### CLI-команда подключения

```bash
claude mcp add supabase -- npx -y @supabase/mcp-server-supabase@latest \
  --project-ref abc123example \
  --access-token sbp_YOUR_TOKEN_HERE
```

### Альтернатива: JSON-конфиг

Добавьте в `.claude/settings.json` проекта:

```json
{
  "mcpServers": {
    "supabase": {
      "command": "npx",
      "args": [
        "-y",
        "@supabase/mcp-server-supabase@latest",
        "--project-ref", "abc123example",
        "--access-token", "sbp_YOUR_TOKEN_HERE"
      ]
    }
  }
}
```

### Проверочный запрос

После подключения выполните в Claude Code:

```
Покажи все таблицы в public-схеме
```

Claude вызовет `list_tables` или выполнит:
```sql
SELECT table_name FROM information_schema.tables WHERE table_schema = 'public';
```

Если вернулся список таблиц — подключение работает.

### Таблица ошибок

| Симптом | Причина | Решение |
|---------|---------|---------|
| "Invalid API key" | Неверный access token | Перегенерировать токен на supabase.com |
| "Project not found" | Неверный project ref | Проверить URL проекта в Dashboard |
| "Connection refused" | Сеть / VPN | Включить VPN (из России) |
| Запросы молча падают | Закэширован старый project ref | Перезапустить сессию Claude Code |
| Auth-цикл (бесконечные ошибки) | Повреждён кэш сессии | Полный перезапуск сессии |

### При смене проекта

**Критично:** при переключении на другой Supabase-проект:
1. Обновите `--project-ref` в конфиге
2. Перезапустите сессию Claude Code
3. Проверьте через `list_tables`

Не надейтесь, что просто обновить ref достаточно — старый ref кэшируется.

## 2. GitHub MCP

### Что делает
Позволяет Claude Code работать с GitHub-репозиториями: создавать PR, просматривать issues, управлять ветками — без ручных команд git в терминале.

### Подготовка
1. GitHub → Settings → Developer Settings → Personal Access Tokens → Fine-grained tokens
2. Создайте токен с правами: `repo`, `read:org`

### CLI-команда подключения

```bash
claude mcp add github -- npx -y @modelcontextprotocol/server-github
```

При этом токен передаётся через переменную окружения.

### JSON-конфиг

```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_YOUR_TOKEN_HERE"
      }
    }
  }
}
```

### Проверочный запрос

```
Покажи последние 5 коммитов в текущем репозитории
```

### Таблица ошибок

| Симптом | Причина | Решение |
|---------|---------|---------|
| "Bad credentials" | Истёк или неверный токен | Перегенерировать PAT на GitHub |
| "Not Found" при доступе к репо | Недостаточно прав токена | Проверить scope токена (нужен `repo`) |
| Медленный ответ | Большой репозиторий | Это нормально, подождать |
| "API rate limit exceeded" | Превышен лимит запросов | Подождать 15 минут или использовать другой токен |

## 3. Context7 MCP

### Что делает
Подтягивает актуальную документацию библиотек (Next.js, Supabase, Tailwind, Zod и др.) прямо в контекст Claude Code. Заменяет ручное гугление документации.

### Почему обязателен
Знания Claude могут устаревать. Context7 гарантирует, что код пишется по актуальной документации — особенно важно для Next.js 16, Tailwind v4, и других активно обновляющихся библиотек.

### CLI-команда подключения

```bash
claude mcp add --transport http context7 https://mcp.context7.com/mcp
```

### JSON-конфиг

```json
{
  "mcpServers": {
    "context7": {
      "type": "url",
      "url": "https://mcp.context7.com/mcp"
    }
  }
}
```

### Проверочный запрос

```
Используя Context7, найди документацию по App Router в Next.js 16
```

### Таблица ошибок

| Симптом | Причина | Решение |
|---------|---------|---------|
| "Connection timeout" | Сервер Context7 недоступен | Попробовать через 5 минут |
| Возвращает устаревшую документацию | Кэш библиотеки | Указать конкретную версию: `resolve-library-id` |
| "Library not found" | Неправильное имя библиотеки | Сначала выполнить `resolve-library-id` для поиска |
| Пустой ответ | Слишком общий запрос | Сузить запрос до конкретной функции/API |

## 4. ChatPlace MCP

### Что делает
Управляет Telegram-ботами, автоматизациями, рассылками и AI-агентами через ChatPlace — прямо из Claude Code. Позволяет создавать воронки, настраивать триггеры и управлять базой знаний бота.

### CLI-команда подключения

```bash
claude mcp add --transport http chatplace https://mcp.chatplace.io/mcp
```

### JSON-конфиг

```json
{
  "mcpServers": {
    "chatplace": {
      "type": "url",
      "url": "https://mcp.chatplace.io/mcp"
    }
  }
}
```

### Проверочный запрос

```
Покажи список ботов в ChatPlace
```

### Таблица ошибок

| Симптом | Причина | Решение |
|---------|---------|---------|
| "Unauthorized" | Не авторизованы в ChatPlace | Войти в аккаунт ChatPlace через браузер |
| Бот не найден | Бот не подключён к ChatPlace | Подключить бота через интерфейс chatplace.io |
| Автоматизация не срабатывает | Неверный триггер | Проверить тип триггера через `automations_get_detail` |
| Медленная отправка | Ограничение Telegram API | Telegram допускает ~30 сообщений/сек |

## 5. NotebookLM MCP

### Что делает
Google NotebookLM — сервис для создания AI-подкастов и аудио-саммари из документации. MCP позволяет загружать документы и генерировать аудио-контент для обучения.

### Подготовка
1. Войдите в Google-аккаунт
2. Откройте NotebookLM: `https://notebooklm.google.com`
3. Настройте API-доступ (если доступен MCP-коннектор)

### Использование
На момент апреля 2026 NotebookLM MCP находится в бета-режиме. Основной способ работы — через веб-интерфейс:
1. Загрузите документацию проекта (спецификацию, CLAUDE.md)
2. Сгенерируйте аудио-обзор для быстрого знакомства с проектом
3. Используйте как обучающий материал для команды

### Альтернативный подход
Если MCP-коннектор недоступен, используйте NotebookLM вручную:
- Загрузите PDF/текст проектной документации
- Попросите сгенерировать Deep Dive (подкаст)
- Используйте как onboarding-материал

## Пример: подключение MCP для проекта TaskFlow

Проект: TaskFlow (трекер задач для фрилансеров)
Компания: ООО "Продуктив"
Supabase ref: `abc123example`

```bash
# Шаг 1: Context7 (глобально, один раз)
claude mcp add --scope user --transport http context7 https://mcp.context7.com/mcp

# Шаг 2: Supabase (для проекта TaskFlow)
claude mcp add supabase -- npx -y @supabase/mcp-server-supabase@latest \
  --project-ref abc123example \
  --access-token sbp_ВАША_СТРОКА_ТОКЕНА

# Шаг 3: GitHub
claude mcp add github -- npx -y @modelcontextprotocol/server-github

# Проверка: все MCP подключены?
claude mcp list
```

После выполнения команд перезапустите сессию Claude Code и проверьте каждый MCP через проверочный запрос из таблиц выше.
